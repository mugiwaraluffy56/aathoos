## What is an ADR?

An Architecture Decision Record captures a decision that has a significant effect on the project: choice of programming language, UI toolkit, storage engine, build system, etc.

## Format

Each ADR is a markdown file named `NNN-short-title.md`:

```
# ADR-001: Choose Rust for core logic

## Status
Accepted / Proposed / Superseded

## Context
What is the issue that we're seeing that is motivating this decision?

## Decision
What is the change that we're proposing and/or doing?

## Consequences
What becomes easier or more difficult because of this change?
```
